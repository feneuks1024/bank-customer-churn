# Прогнозирование оттока клиентов банка

Проект посвящён построению и сравнению моделей машинного обучения для прогнозирования оттока клиентов банка. Задача решается как бинарная классификация: модель должна определить, уйдёт клиент из банка или останется.

Основной акцент работы сделан на повышении **Precision** для класса ушедших клиентов без чрезмерного снижения **Recall**. Для этого используются мягкая балансировка классов и подбор порога классификации на отдельной validation-выборке.

## Цель проекта

* подготовить данные для обучения моделей;
* исследовать влияние дисбаланса классов;
* обучить несколько алгоритмов классификации;
* сравнить модели по Accuracy, Precision, Recall, F1-score и ROC-AUC;
* подобрать индивидуальный порог классификации для каждой модели;
* выбрать лучшую модель для прогнозирования оттока клиентов.

## Датасет

В работе используется таблица из **10 000 строк и 18 столбцов**.

Целевая переменная:

* `Exited = 0` — клиент остался в банке;
* `Exited = 1` — клиент ушёл из банка.

Распределение классов:

| Класс          |  Доля |
| -------------- | ----: |
| Клиент остался | 79,6% |
| Клиент ушёл    | 20,4% |

Датасет имеет заметный дисбаланс классов, поэтому Accuracy сама по себе не является достаточной метрикой качества.

Из обучения исключаются признаки, которые не несут полезной информации либо могут создавать утечку данных:

* `RowNumber`;
* `CustomerId`;
* `Surname`;
* `Complain`;
* `Satisfaction Score`;
* `Card Type`;
* `Point Earned`.

После удаления служебных столбцов используются следующие признаки:

* `CreditScore`;
* `Geography`;
* `Gender`;
* `Age`;
* `Tenure`;
* `Balance`;
* `NumOfProducts`;
* `HasCrCard`;
* `IsActiveMember`;
* `EstimatedSalary`.

## Предобработка данных

В ноутбуке выполняются следующие этапы:

1. Разделение исходных данных на train и test в пропорции 80/20 со стратификацией по целевой переменной.
2. Разделение train-части на обучающую и validation-выборки.
3. Удаление идентификаторов и потенциально некорректных признаков.
4. Заполнение пропусков:

   * медианой для числовых признаков;
   * наиболее частым значением для категориальных признаков.
5. One-Hot Encoding категориальных признаков `Geography` и `Gender`.
6. Масштабирование данных для Logistic Regression с помощью `StandardScaler`.
7. Сохранение одинакового набора признаков в train, validation и test.

Итоговый размер выборок после предобработки:

| Выборка    |    Размер |
| ---------- | --------: |
| Train      | 6400 × 11 |
| Validation | 1600 × 11 |
| Test       | 2000 × 11 |

## Используемые модели

В работе сравниваются шесть алгоритмов:

* Logistic Regression;
* Random Forest;
* Gradient Boosting;
* XGBoost;
* LightGBM;
* CatBoost.

Для моделей, поддерживающих веса классов, применяется мягкая балансировка. Вместо полного веса положительного класса используется более умеренное значение, чтобы не повышать Recall ценой резкого падения Precision.

## Подбор порога классификации

Стандартный порог бинарной классификации равен `0.5`. Однако при несбалансированных классах он не всегда обеспечивает подходящее соотношение Precision и Recall.

В проекте для каждой модели перебираются пороги от `0.30` до `0.80` с шагом `0.01`. Порог выбирается на validation-выборке по следующим правилам:

1. `Recall` должен быть не ниже `0.55`;
2. среди подходящих вариантов максимизируется `Precision`;
3. при равенстве учитываются `F1-score` и `Accuracy`.

Test-выборка не используется при подборе порога и применяется только для итоговой оценки.

### Лучшая модель

По итогам эксперимента лучшей моделью является **CatBoost**.

Её показатели:

* порог классификации — `0.60`;
* Accuracy — `0.8650`;
* Precision — `0.7104`;
* Recall — `0.5711`;
* F1-score — `0.6332`;
* ROC-AUC — `0.8779`.

После изменения порога Precision CatBoost увеличился с `0.6402` до `0.7104`, при этом F1-score снизился незначительно: с `0.6363` до `0.6332`.

## Основной вывод

Низкий Precision у части моделей связан не только с дисбалансом классов, но и с выбором весов классов и стандартного порога `0.5`. Слишком сильная балансировка повышает Recall, однако может приводить к большому числу ложноположительных прогнозов.

Мягкая балансировка вместе с подбором порога позволяет уменьшить количество клиентов, ошибочно отнесённых к ушедшим. В данном эксперименте лучший компромисс между Precision, Recall и F1-score показала модель CatBoost с порогом `0.60`.

## Ограничения

* Гиперпараметры моделей подбираются вручную, без полноценного Grid Search или Bayesian Optimization.
* Порог оптимизируется под условие `Recall >= 0.55`; при других бизнес-требованиях оптимальный порог может измениться.
* Для применения модели в реальной банковской системе потребуется проверка на новых данных и анализ изменения распределения признаков во времени.
